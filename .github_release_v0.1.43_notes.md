5. Expanded runtime regression and release consistency gates:
   - `audit-fail-missing-presets`

## 繁體中文
### 重點更新
1. `gov_audit` 由模板式 PASS 改為可執行的 12 項 QC 逐條判定。
2. 新增逐項 QC 輸出：
   - `qc_items`（每項 PASS/FAIL/PASS(N/A) + 詳細原因）
   - `qc_summary`（pass/fail/pass_na 計數）
   - audit 失敗時在指令輸出顯示 `qc_failed_items`
3. 保留固定分母政策（`12/12`），同時移除「假完成」行為。
4. Audit runner 補強決定性證據檢查：
   - 必需讀檔證據
   - marker 唯一性
   - index 更新驗證
   - root 治理溢出檢測
   - 若存在最近寫入 run，驗證 before/after 備份證據
5. 擴展 runtime regression 與 release consistency gate：
   - `audit-fail-missing-presets`
   - `audit-pass-12-items`

### 驗證結果
1. `node dev/run_runtime_regression.mjs` -> `SUMMARY 30/30 passed`
2. `node dev/check_release_consistency.mjs` -> `ALL_CHECKS_PASS`

### 升級步驟
1. `openclaw plugins update openclaw-workspace-governance`
2. `openclaw gateway restart`
3. 在 TUI 執行：`/gov_setup check`
4. 繼續：`/gov_migrate` -> `/gov_audit`
