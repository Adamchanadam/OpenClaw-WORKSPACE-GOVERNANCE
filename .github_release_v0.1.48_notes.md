# v0.1.48

## Bug Fix: Audit false-failure after `/gov_brain_audit APPROVE`

### Problem
After `/gov_brain_audit APPROVE: F001,F002` applied changes, running `/gov_audit` could fail with QC 8 (BEFORE/AFTER PROOF) and QC 3 (INDEX UPDATED) errors. The root cause was that `findLatestWriteRunReport()` picked up LLM-generated brain audit run reports that lack the deterministic metadata format expected by QC checks.

### Fix
Changed `findLatestWriteRunReport()` filter from blacklist (`!/^gov_audit_/i`) to whitelist (`WRITE_RUN_REPORT_NAME_RE`). Only known deterministic write reports (`migrate_governance_rev6_*`, `gov_uninstall_*`, `*_apply_upgrade_from_boot_v1*`) are now considered. All LLM-generated reports are automatically excluded.

### Regression
- New test: `audit-ignores-non-deterministic-run-reports`
- Baseline: `35/35 passed` (was 34/34)
- New Phase B6 scenario in `OPENCLAW_PUBLIC_FLOW_REGRESSION.md`

### Upgrade
```
openclaw plugins update openclaw-workspace-governance
openclaw gateway restart
/gov_audit
```
