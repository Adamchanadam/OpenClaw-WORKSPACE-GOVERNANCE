# v0.1.54 — Fix brain_audit false positives on read-only run reports

## Key Changes

- **Fixed `COMPLETION_WITHOUT_EVIDENCE` false positives**: Read-only audit run reports (`gov_audit_*`, `gov_brain_audit_preview_*`, `gov_boot_audit_scan_*`) no longer trigger spurious HIGH findings. These reports correctly omit `TARGET_FILES_TO_CHANGE` because audit/preview/scan operations don't modify files.
- **Improved APPROVE guidance**: WARN/BLOCKED next-step suggestions now include actual finding IDs (e.g., `/gov_brain_audit APPROVE: F001,F002,F003`) instead of a hardcoded example `F001`.
- **Regression**: 104/104 (C104 added for read-only report exemption)

## Practical Impact

- Runtimes upgrading from v0.1.53 will no longer see 5 spurious HIGH findings from their existing audit run reports
- Operators can directly copy-paste the suggested APPROVE command with correct finding IDs
- No breaking changes — write-operation reports still require full evidence fields

# v0.1.54 — 修正 brain_audit 對唯讀 run reports 的誤報

## 關鍵變更

- **修正 `COMPLETION_WITHOUT_EVIDENCE` 誤報**：唯讀 audit run reports（`gov_audit_*`、`gov_brain_audit_preview_*`、`gov_boot_audit_scan_*`）不再觸發虛假 HIGH findings。這些報告正確地省略 `TARGET_FILES_TO_CHANGE`，因為 audit/preview/scan 操作不修改檔案。
- **改進 APPROVE 指引**：WARN/BLOCKED 下一步建議現在包含實際 finding IDs（例如 `/gov_brain_audit APPROVE: F001,F002,F003`），而非固定的範例 `F001`。
- **回歸測試**：104/104（新增 C104 唯讀報告豁免測試）

## 對使用者的直接影響

- 從 v0.1.53 升級的 runtime 不再看到來自現有 audit run reports 的 5 個虛假 HIGH findings
- 操作者可直接複製建議的 APPROVE 指令（包含正確的 finding IDs）
- 無破壞性變更 — 寫入操作報告仍需完整的證據欄位
