# v0.1.55 — Pre-Modification Config Reference Verification

## What's New

### `/gov_openclaw_json` — Config Reference Scan (v0.1.55)

The runner now scans the local workspace for OpenClaw documentation **before** any `openclaw.json` modification. The SKILL layer adds a three-tier verification flow:

1. **Local docs first** — scans `skills/` directory for `openclaw-doc` or config reference materials
2. **Official web fallback** — reads `docs.openclaw.ai`, GitHub releases, and GitHub repo when local docs are insufficient
3. **Graceful degradation** — if web fetch is unavailable, proceeds with operator confirmation; never deadlocks

Output now includes `CONFIG_REF_SOURCE` evidence field (`local docs` / `official web` / `operator-confirmed` / `unverified`).

### Runner Enhancement
- New `scanLocalConfigReference()` in `gov_openclaw_json_sync.mjs` — deterministic BFS scan of `skills/` directory
- `configRefScan` field added to command output and run reports (`## CONFIG_REF_SCAN` section)

### Regression & Quality
- Regression baseline: **108/108** (was 104/104)
- 4 new test cases: C105 (field presence), C106 (local doc detection), C107 (missing skills/ dir), C108 (run report section)
- 2 new release consistency policy checks (SKILL.md contract + index.ts wiring guard)
- Canonical bootstrap payload synced

# v0.1.55 — 預修改設定參考驗證

## 新功能

### `/gov_openclaw_json` — 設定參考掃描 (v0.1.55)

Runner 現在會在任何 `openclaw.json` 修改**之前**掃描本地工作區的 OpenClaw 文檔。SKILL 層新增三層驗證流程：

1. **本地文檔優先** — 掃描 `skills/` 目錄尋找 `openclaw-doc` 或設定參考資料
2. **官網讀取備援** — 本地文檔不足時讀取 `docs.openclaw.ai`、GitHub releases 和 GitHub repo
3. **優雅降級** — 若無 web fetch 功能，以操作員確認方式繼續；絕不會卡死

輸出新增 `CONFIG_REF_SOURCE` 證據欄位（`local docs` / `official web` / `operator-confirmed` / `unverified`）。

### Runner 增強
- 新增 `scanLocalConfigReference()` 於 `gov_openclaw_json_sync.mjs` — 確定性 BFS 掃描 `skills/` 目錄
- 命令輸出及 run report 新增 `configRefScan` 欄位（`## CONFIG_REF_SCAN` section）

### 回歸與品質
- 回歸基線：**108/108**（原 104/104）
- 4 個新測試案例：C105（欄位存在）、C106（本地文檔偵測）、C107（無 skills/ 目錄）、C108（run report section）
- 2 個新發佈一致性政策檢查（SKILL.md 合約 + index.ts 接線守衛）
- 規範引導源檔案 payload 已同步
