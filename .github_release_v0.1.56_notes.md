# v0.1.56 — Advisory-First UX Reform

## What's New

### Advisory-First Write Gate Model

The governance runtime no longer demands machine tokens (`WG_PLAN_GATE_OK` / `WG_READ_GATE_OK`) from users. The AI follows PLAN→READ→CHANGE→QC→PERSIST **internally** and proceeds directly with user tasks.

**Risk-tiered write protection:**

| Target Type | Behavior |
|---|---|
| Normal writes (skills/, projects/, code) | Always advisory — never hard-blocked |
| High-risk writes (Brain Docs, openclaw.json) | Advisory 1st–2nd attempt, hard block 3rd+ |
| 3+ consecutive blocks | Escape hint with `/gov_brain_audit force-accept` |

### Separate High-Risk Counter

Normal advisory writes no longer inflate the high-risk blocking threshold. The `highRiskBlockedWrites` counter is independent — high-risk targets always get their full 2-advisory window regardless of prior normal writes.

### Natural-Language Evidence Detection

Plan and read evidence is now detected from natural prose in English and Chinese:
- Plan: "Plan:", "Objective:", "I will create...", "策略：", "Steps:"
- Read: "I read the files", "files reviewed:", "已讀", "existing content"

### Shell Command False-Positive Reduction

`extractTargetPath` now strips quoted string contents for shell commands, preventing incidental mentions (e.g., `echo "check AGENTS.md" >> log.txt`) from triggering high-risk classification.

### Aligned Payloads

- AGENTS.md payload: PLAN-first and No-Write rules changed from "Hard" to "Best Practice"
- GOVERNANCE_BOOTSTRAP.md payload: aligned to match AGENTS.md reform
- Gate tokens documented as "advisory internal signals" — users never type them
- `prependContext` says "Proceed directly with the user's task"
- Block messages use human-readable language, no machine tokens

### Regression & Quality
- Regression baseline: **135/135** (was 104/104 in v0.1.54)
- Code review findings: 3 CRITICAL + 4 WARNING items identified and fixed
- Risk Classification tables added to both READMEs

# v0.1.56 — 建議優先 UX 改革

## 新功能

### 建議優先寫入閘門模型

治理運行時不再要求用戶輸入機器令牌（`WG_PLAN_GATE_OK` / `WG_READ_GATE_OK`）。AI 在**內部**遵循 PLAN→READ→CHANGE→QC→PERSIST 流程，直接處理用戶任務。

**風險分層寫入保護：**

| 目標類型 | 行為 |
|---|---|
| 普通寫入（skills/、projects/、代碼） | 永遠僅建議 — 絕不硬封鎖 |
| 高風險寫入（Brain Docs、openclaw.json） | 第 1-2 次建議，第 3 次起硬封鎖 |
| 連續 3+ 次封鎖 | 顯示逃生提示 `/gov_brain_audit force-accept` |

### 獨立高風險計數器

普通建議性寫入不再膨脹高風險封鎖閾值。`highRiskBlockedWrites` 計數器獨立運作 — 高風險目標無論之前有多少普通寫入，始終享有完整的 2 次建議窗口。

### 自然語言證據偵測

現在可從英文和中文的自然語句中偵測計劃和讀取證據：
- 計劃："Plan:"、"Objective:"、"I will create..."、"策略："、"Steps:"
- 讀取："I read the files"、"files reviewed:"、"已讀"、"existing content"

### Shell 命令誤報減少

`extractTargetPath` 現在會剝除 shell 命令中的引號內容，防止偶然提及（如 `echo "check AGENTS.md" >> log.txt`）觸發高風險分類。

### 對齊的 Payload

- AGENTS.md payload：PLAN-first 和 No-Write 規則從「Hard」改為「Best Practice」
- GOVERNANCE_BOOTSTRAP.md payload：已對齊匹配 AGENTS.md 改革
- Gate tokens 記錄為「建議性內部信號」— 用戶無需輸入
- `prependContext` 說「直接處理用戶的任務」
- 封鎖訊息使用人類可讀語言，無機器令牌

### 回歸與品質
- 回歸基線：**135/135**（v0.1.54 為 104/104）
- 代碼審查發現：3 個 CRITICAL + 4 個 WARNING 項目已識別並修復
- 兩個 README 新增風險分類表
