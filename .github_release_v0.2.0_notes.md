# v0.2.0 — Governance Hook Resilience

## 為什麼需要這個更新 / Why This Update Was Needed

OpenClaw v2026.3.x introduced architectural changes that created governance bypass vectors:

1. **v2026.3.1 — Cron/heartbeat light-context mode** (`heartbeat.lightContext: true` / cron `--light-context`): skips AGENTS.md bootstrap injection entirely. The plugin's event-driven `prependContext` only fires when specific conditions match — but if bootstrap is never injected, the Mode A/B/C anchor never reaches the agent.

2. **v2026.3.7 — `postCompactionSections` config**: allows operators to exclude the governance section from post-compaction context. If the user configures this, governance directives can disappear from context mid-conversation.

3. **Combined effect**: automated cron/heartbeat runs could operate with **zero governance context** if bootstrap injection was skipped.

The root fix: move the static Mode A/B/C governance anchor from AGENTS.md bootstrap dependency → plugin hook mechanism (`prependSystemContext`), which fires **unconditionally every turn at the plugin layer**, below bootstrap level.

---

## 改了什麼 / What Changed

### Core: `prependSystemContext` Always-On Governance Anchor

New `buildGovernanceSystemAnchor()` function in `index.ts` injects a compact governance summary into **system prompt space** every turn via the new `prependSystemContext` field (OpenClaw v2026.3.7+ API):

```
[OpenClaw Governance v0.2.0 — active] Execution order: PLAN → READ → CHANGE → QC → PERSIST.
Before any write/edit/save: include PLAN (what + why) and READ (files read).
Mode A = conversation only | B = read + verify sources | C = write + evidence required.
```

All 9 return paths in the `before_prompt_build` handler now include `prependSystemContext`. The anchor is **provider-cacheable** (system prompt space) — lower per-turn token cost than `prependContext`.

### Automation-Aware Notice for Cron/Heartbeat Runs

When `ctx.trigger` is `"cron"` or `"heartbeat"`, the anchor appends:

```
(Automated cron run — bootstrap injection may be skipped; this governance anchor is always active.)
```

This ensures automated runs always know governance is active, even without AGENTS.md bootstrap.

### `sessionId`-Based Gate State Isolation

`toSessionKey()` now prefers `ctx.sessionId` (ephemeral UUID, regenerated on `/new` and `/reset`) over `ctx.sessionKey`. This means:
- Gate state (blocked write counters, advisory flags, quiet-turn tracking) correctly resets when the user starts a new conversation
- Two sessions with the same routing `sessionKey` but different `sessionId` values get independent state — no cross-conversation gate state bleed

Backward-compatible: falls back to `sessionKey` on runtimes that pre-date `sessionId` support.

### Type Definitions Updated

`types/openclaw-plugin-sdk.d.ts` updated with all OpenClaw v2026.3.7+ new API fields:
- `PluginHookAgentContext`: `sessionId`, `trigger`, `channelId`
- `PluginHookToolContext`: `sessionId`, `runId`, `toolCallId`
- `PluginHookBeforePromptBuildResult`: `prependSystemContext`, `appendSystemContext`
- `PluginHookBeforeToolCallEvent`: `runId`, `toolCallId`

## 繁體中文版更新說明

### 為什麼更新

OpenClaw v2026.3.x 引入的架構變更製造了治理繞過漏洞：

1. **v2026.3.1 Cron/heartbeat light-context 模式**：跳過 AGENTS.md bootstrap 注入，治理指令無法到達 agent
2. **v2026.3.7 `postCompactionSections` 配置**：用戶可配置壓縮後排除治理段落，治理指令可能在對話中途消失
3. **綜合影響**：自動化 cron/heartbeat 執行可能在完全無治理上下文的情況下運行

根本修復：將靜態 Mode A/B/C 治理錨點從依賴 AGENTS.md bootstrap → 移至 plugin hook 機制（`prependSystemContext`），在 plugin 層無條件每回合觸發，低於 bootstrap 層級。

### 主要改動

- `buildGovernanceSystemAnchor()` 函數：每回合透過 `prependSystemContext` 注入系統提示空間（provider 可緩存，低 token 成本）
- cron/heartbeat 觸發時加入自動化提示，確保自動化執行知道治理始終生效
- `toSessionKey()` 優先使用 `sessionId`（每次 `/new` 和 `/reset` 後重新生成），確保閘門狀態按 session 正確隔離
- 類型定義更新至 OpenClaw v2026.3.7+ 全新 API 欄位
- 10 個新驗收測試涵蓋所有 9 個 `before_prompt_build` 返回路徑
